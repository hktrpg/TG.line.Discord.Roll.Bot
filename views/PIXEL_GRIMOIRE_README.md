# 🎭 Pixel Grimoire - 像素魔法體驗

## 📖 概述

Pixel Grimoire 是 Grimoire.OS 項目的原型實現，提供了一個像素藝術風格的 TRPG 遊戲介面，展示基本的網格佈局和動態像素精靈。

## 🌟 主要功能

### 🖼️ 像素藝術介面
- **三區域佈局**: 左側導航欄 + 上方狀態BAR + 右側內容區域
- **可伸縮導航**: 點擊按鈕收起/展開導航欄
- **網格背景**: 32x32 像素的網格系統
- **像素主題**: 使用精心設計的像素藝術色彩調色盤

### 📊 即時狀態BAR
- **點陣圖風格**: 像素藝術風格的數位顯示
- **動態人數統計**: 線上人數、活躍精靈數量
- **房間狀態**: 根據活躍度顯示不同狀態
- **系統資訊**: 即時更新系統負載和活動時間

### 🎮 精靈動畫系統
- **多種類型精靈**: GM、玩家、NPC 三種不同風格的精靈
- **即時動畫**: 行走動畫、活躍狀態指示
- **互動式移動**: 點擊精靈或使用鍵盤控制移動

### 🎯 互動功能
- **雙模式體驗**:
  - **觀察模式**: WASD移動，點擊精靈自由移動
  - **遊戲模式**: ↑鍵跳躍，→鍵攻擊，空格切換模式
- **障礙物挑戰**: 牆壁、尖刺、石塊等需要跳過或打破
- **即時碰撞**: 與障礙物互動會有視覺回饋
- **計分系統**: 打破障礙物得分，碰到尖刺扣分

### 🧭 導航系統
- **可伸縮導航欄**: 點擊右上角按鈕收起/展開
- **功能模組**: 儀表板、跑團管理、角色卡、擲骰等
- **視覺指示**: 活躍項目高亮顯示

### 📊 狀態監控BAR
- **純像素視覺效果**: 只有掃描線動畫，無文字顯示
- **藝術裝飾元素**: 提供視覺節奏感

### 🎨 UI 組件
- **左側導航**: 可伸縮的功能導航欄
- **上方狀態BAR**: 像素精靈活躍的遊戲舞台
  - 觀察模式：自由移動和互動
  - 遊戲模式：障礙物挑戰和計分
- **中央遊戲區域**: 空的視覺空間（預留擴展）
- **底部工具欄**: 功能按鈕區域

## 🎮 使用方法

### 基本操作
1. **啟動伺服器**: `node index.js`
2. **訪問頁面**: `http://localhost:20721/pixel-grimoire`
3. **導航控制**:
   - 點擊導航欄右上角按鈕收起/展開導航欄
   - 點擊不同功能模組切換活躍狀態
4. **雙模式遊戲**:
   - **觀察模式**: WASD移動，點擊精靈自由移動
   - **遊戲模式**: 按空格鍵進入，↑跳躍，→攻擊
5. **遊戲挑戰**:
   - 避開尖刺（紅色三角形）
   - 打破牆壁和石塊（攻擊時碰撞）
   - 累積最高分數

### 佈局特點
- **左側導航欄**: 可伸縮，包含各功能模組
- **上方狀態BAR**: 點陣圖風格的即時統計
- **右側內容區域**: 遊戲區域和控制面板
- **底部工具欄**: 功能按鈕區域

## 🛠️ 技術實現

### CSS 架構
```css
:root {
  --pixel-scale: 2;
  --grid-size: 32px;
  --palette: {
    background: #2d1b69,
    primary: #ff6b6b,
    secondary: #4ecdc4,
    accent: #ffe66d
  }
}
```

### JavaScript 動畫系統
- **SpriteManager 類**: 管理所有精靈的狀態和動畫
- **平滑移動**: 使用 requestAnimationFrame 的精靈移動
- **碰撞檢測**: 基礎的邊界檢查
- **事件處理**: 鍵盤和滑鼠事件響應

### 像素藝術設計原則
- **統一縮放**: 所有元素使用 2x 像素縮放
- **色彩調色盤**: 限定的色彩確保視覺一致性
- **網格對齊**: 所有元素對齊到像素網格
- **簡潔造型**: 使用簡單的幾何形狀和鮮明輪廓
